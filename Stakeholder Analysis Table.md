
# Stakeholder Analysis

| Stakeholder | Role | Key Concerns | Pain Points | Success Metrics |
|------------|------|-------------|------------|----------------|
| Gym Members | Use the system to register, book classes, and manage memberships | Easy booking, quick access to schedules, secure payments | Long queues, booking conflicts, no real-time updates | 30% faster booking process, real-time class availability |
| Gym Administrators | Manage operations, members, and system data | Efficient management, accurate reports, centralized control | Manual errors, scattered data systems | Reduced errors, improved operational efficiency |
| Trainers | Conduct classes and manage schedules | Clear schedules, proper class assignments | Double bookings, unclear schedules | Accurate scheduling, no conflicts |
| Reception Staff | Handle registrations and customer queries | Fast member registration, easy system use | Time-consuming manual entry | Reduced registration time |
| IT Staff | Maintain system infrastructure and performance | System stability, security, easy maintenance | System crashes, difficult updates | High system uptime (99%), quick issue resolution |
| Gym Management (Owners) | Oversee business performance and growth | Business insights, revenue tracking | Lack of data for decision-making | Improved revenue tracking, data-driven decisions |
| Payment Service Providers | Handle payment processing | Secure and reliable transactions | Payment failures, security risks | 99% successful transactions |

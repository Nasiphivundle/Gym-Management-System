

## Use Case: Create Task

Actor: Student

Description: Allows student to create a new task

Preconditions:
- User must be logged in

Postconditions:
- Task is saved in the system

Basic Flow:
1. Student clicks "Create Task"
2. Student enters task details
3. Student clicks save
4. System stores task

Alternative Flow:
- If fields are empty → show error message




## Use Case: Login

Actor: Student

Preconditions:
- User must be registered

Postconditions:
- User is logged into system

Basic Flow:
1. Enter username and password
2. Click login
3. System validates credentials

Alternative Flow:
- Invalid login → error message
